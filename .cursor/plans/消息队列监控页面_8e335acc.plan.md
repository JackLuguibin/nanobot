## 实现步骤

### 阶段 1: 后端扩展

#### 1.1 扩展 `console/server/extension/zmq_bus.py` — 新增状态获取方法

在 `ZeroMQBus` 类中添加以下方法：

- `get_queue_status()` → 返回所有 ZMQ Socket 连接状态（PUB/ROUTER/SUB 列表）
- `get_pending_messages()` → 返回待处理消息数
- `get_recent_messages(limit=100)` → 返回最近消息记录（可选，用于历史展示）

同时在文件底部添加模块级函数 `get_queue_status()` 方便 API 层调用。

#### 1.2 扩展 `console/server/api/state.py` — 暴露队列状态

在 `BotState` 类中添加：

- `get_queue_status()` → 调用 `self._agent_manager.get_queue_status()` 和 `MessageBus` 的 `inbound_size`/`outbound_size`

在 `BotStateManager` 中添加聚合方法 `get_all_queue_status()` 汇总所有 Bot 的队列状态。

#### 1.3 新建 `console/server/api/queue.py` — 队列专用 API

```python
# GET /api/queue/status          - 获取所有队列状态
# GET /api/queue/status/{bot_id} - 获取单个 Bot 队列状态
# WebSocket: 广播 queue_update 消息（定时推送，每 3 秒）
```

#### 1.4 注册路由 `console/server/api/routes.py`

在 `router` 中 include `queue_router`（路径: `/api/queue`）。

#### 1.5 WebSocket 推送 `console/server/api/websocket.py`

在 `ConnectionManager` 中添加 `broadcast_queue_update()` 方法，WebSocket 连接后定期（3 秒间隔）推送队列状态给订阅者。

### 阶段 2: 前端新增页面

#### 2.1 新建页面 `console/web/src/pages/Queue.tsx`

页面布局（使用 Ant Design 组件）：

```
┌─────────────────────────────────────────────────────┐
│  页面标题: 消息队列监控              [刷新] [自动刷新✓] │
├───────────────────────┬─────────────────────────────┤
│  Channel 层 (asyncio.Queue)                         │
│  ┌────────────────┐  ┌────────────────┐              │
│  │ Inbound 队列   │  │ Outbound 队列  │              │
│  │ 待处理: 3      │  │ 待处理: 1       │              │
│  └────────────────┘  └────────────────┘              │
├─────────────────────────────────────────────────────┤
│  Agent 层 (ZeroMQ Bus)                              │
│  ┌──────────────┐  ┌──────────────┐               │
│  │ PUB Socket   │  │ ROUTER Socket │               │
│  │ tcp://...    │  │ tcp://...     │               │
│  └──────────────┘  └──────────────┘               │
│  订阅 Agent 列表:                                   │
│  ┌──────────────────────────────────────────────┐  │
│  │ Agent ID    │ Bot ID │ Topics │ 连接状态      │  │
│  │ bot_a:agent1│ bot_a  │ [msg]  │ ✓ 连接       │  │
│  └──────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

- 使用 `antd Card` 展示各队列状态
- 使用 `antd Table` 展示订阅 Agent 列表
- 使用 `antd Statistic` 展示队列大小
- 颜色指示：队列空时绿色，有消息时橙色/红色
- 自动刷新（默认开启，3 秒间隔），可手动关闭

#### 2.2 添加类型定义 `console/web/src/api/types_queue.ts` [新增]

```typescript
export interface ChannelQueueStatus {
  inbound_size: number;
  outbound_size: number;
}

export interface ZMQSocketInfo {
  type: 'PUB' | 'ROUTER' | 'SUB';
  address: string;
}

export interface ZMQQueueStatus {
  pub_socket: ZMQSocketInfo | null;
  router_socket: ZMQSocketInfo | null;
  sub_sockets: { agent_id: string; bot_id: string; topics: string[]; address: string }[];
  pending_delegations: number;
}

export interface QueueStatus {
  bot_id: string;
  channel_queue: ChannelQueueStatus;
  zmq_queue: ZMQQueueStatus;
  last_updated: string;
}
```
